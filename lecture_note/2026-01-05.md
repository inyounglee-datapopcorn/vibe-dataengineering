## 📚 상세 학습 내용

### 1️⃣ Git & GitHub 협업 프로세스
*   **파일 상태(Status) 가이드:**
    *   `A` (Added): 새롭게 생성되어 추적 시작된 파일.
    *   `M` (Modified): 기존 파일 중 수정이 발생한 파일.
    *   `D` (Deleted): 삭제된 파일.
    *   `U` (Untracked): Git이 아직 추적하지 않는 파일.
*   **`.gitignore` 활용:** `DS_Store`나 로그 파일 등 레포지토리에 포함되지 않아야 할 불필요한 파일을 정의하는 설정 파일.
*   **실무 팁:** 수강생은 강사님의 업데이트 자료를 받기 위해 **Pull** 기능을 주로 사용하며, 전체 변경 이력은 Source Control 탭의 그래프로 시각화하여 확인 가능.

### 2️⃣ Antigravity (AI-Native IDE) 심화
*   **아키텍처:** VS Code 기반으로 구축되어 익숙한 UI를 제공하면서도, Google Gemini 모델이 에디터 전반에 밀접하게 통합된 구조.
*   **핵심 컴포넌트:**
    *   **Agent:** 코드 작성 및 웹 검색을 자율적으로 수행하는 가상 직원.
    *   **Artifacts:** 작업 결과물을 즉시 렌더링하여 보여주는 미리보기 기능.
    *   **Agent Manager:** 복수의 에이전트를 생성하고 관리하는 제어 센터.
*   **모드 설정:** 
    *   `Fast Mode`: 즉각적인 응답이 필요한 간단한 작업에 권장.
    *   `Planning Mode`: 복잡한 로직 설계 시 사용(토큰 소모량 높음).

### 3️⃣ AI 협업 워크플로우 (과업 지시 3단계)
1.  **목표 설정 (Goal Setting):** 만들고자 하는 서비스나 기능의 상위 목표 정의.
2.  **작업 지시 (Instruction):** 환경, 범위, 제한 사항 등을 포함한 구체적인 프롬프트 작성.
3.  **검토 및 피드백 (Review):** AI의 결과물을 검증하고 지속적으로 개선 요청.

### 4️⃣ 데이터 직군 트렌드 및 역할 변화
*   **변화하는 경계:** AI 기술 장벽 하락으로 인해 `Analytics Engineer`와 같이 여러 직군의 역량을 아우르는 하이브리드 역할 부상.
*   **직군별 역할 비유 (건축):**
    *   **DE (Data Engineer):** 파이프라인 및 인프라 구축 (토목 및 뼈대 공사).
    *   **DA (Data Analyst):** 데이터 기반 인사이트 도출 및 시각화 (내부 인테리어).
    *   **DS (Data Scientist):** 머신러닝 모델링 및 예측 (특수 설계).
*   **핵심 역량:** 도메인 지식 이상으로 데이터에서 이상치를 발견하고 **"왜?"**라는 질문을 던져 비즈니스 임팩트를 만들어내는 **이슈 제기(Issue Raising)** 능력이 최우선.

## 📌 강의 핵심 요약
오전의 마지막 세션에서는 실무 데이터 분석가(DA)가 갖춰야 할 **비즈니스 이해도**와 **통계적 유의성 해석의 실제**를 다루었습니다. 특히 AI 시대를 맞아 데이터 엔지니어링과 분석을 융합한 **Analytics Engineer(AE)** 직무의 중요성을 강조하고, 신입이 갖춰야 할 '넓고 얕은' 지식 습득 전략인 **'다트(Dart) 이론'**을 제시했습니다.

## 📚 상세 학습 내용

### 1️⃣ 데이터 분석가(DA)의 역량과 태도
*   **비즈니스 이해도:** 단순 기술(SQL/Python) 숙련도를 넘어, 수익 구조와 비용 절감 등 비즈니스 모델에 대한 깊은 이해 필수.
*   **리서치 역량:** 도메인 지식 보완을 위해 관련 법규 및 해외 선도 사례(Uber, Didi 등)를 지속적으로 분석해야 함.
*   **직관(Intuition) 재해석:** 경영진의 직관은 경험적 데이터의 축적물임을 이해하고, 데이터가 모든 의사결정의 유일한 정답이 아님을 인지.
*   **통계의 실무적 적용:** A/B 테스트의 통계적 유의성이 반드시 비즈니스 성공으로 직결되지는 않음. 분석가는 수치를 맹신하기보다 현실과의 괴리를 논리적으로 방어할 수 있어야 함.

### 2️⃣ Analytics Engineer(AE)의 역할과 가치
*   **직군 융합 트렌드:** AI 기반 자동화로 인해 DA, DE, DS 간 기술적 경계가 모호해지며 조직이 소수 정예화됨.
*   **AE의 정의:** 데이터 엔지니어링(인프라 안정성)과 데이터 분석(비즈니스 인사이트)을 연결하는 가교.
*   **주요 과업:** 데이터 중앙화(Centralization), 데이터 거넌스(Governance) 관리, 전사적 지표 표준화 등.
*   **생존 전략:** AI로 대체하기 어려운 **인프라(DE) 역량**을 토대로 분석력을 갖춘 AE가 시장에서 가장 높은 경쟁력을 가짐.

### 3️⃣ 신입의 학습 전략: '다트(Dart) 이론'
*   **광범위한 개념 습득 (Broad Knowledge):** 신입에게 가장 치명적인 것은 '기술의 존재 자체를 모르는 것'. 최대한 많은 키워드(ETL, EC2, AWS 등)를 얕더라도 넓게 파악해야 함.
*   **전이 가능한 학습:** 하나의 도구(예: MySQL)를 깊게 익히면 유사한 상위 기술(Snowflake, BigQuery 등) 습득 속도가 비약적으로 빨라짐.
*   **AI 활용:** AI를 학습 도구로 삼아 지식 습득의 임계값을 빠르게 넘기는 효율적 학습 권장.
