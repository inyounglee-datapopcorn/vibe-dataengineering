## 📚 상세 학습 내용

### 1️⃣ Git & GitHub 협업 프로세스
*   **파일 상태(Status) 가이드:**
    *   `A` (Added): 새롭게 생성되어 추적 시작된 파일.
    *   `M` (Modified): 기존 파일 중 수정이 발생한 파일.
    *   `D` (Deleted): 삭제된 파일.
    *   `U` (Untracked): Git이 아직 추적하지 않는 파일.
*   **`.gitignore` 활용:** `DS_Store`나 로그 파일 등 레포지토리에 포함되지 않아야 할 불필요한 파일을 정의하는 설정 파일.
*   **실무 팁:** 수강생은 강사님의 업데이트 자료를 받기 위해 **Pull** 기능을 주로 사용하며, 전체 변경 이력은 Source Control 탭의 그래프로 시각화하여 확인 가능.

### 2️⃣ Antigravity (AI-Native IDE) 심화
*   **아키텍처:** VS Code 기반으로 구축되어 익숙한 UI를 제공하면서도, Google Gemini 모델이 에디터 전반에 밀접하게 통합된 구조.
*   **핵심 컴포넌트:**
    *   **Agent:** 코드 작성 및 웹 검색을 자율적으로 수행하는 가상 직원.
    *   **Artifacts:** 작업 결과물을 즉시 렌더링하여 보여주는 미리보기 기능.
    *   **Agent Manager:** 복수의 에이전트를 생성하고 관리하는 제어 센터.
*   **모드 설정:** 
    *   `Fast Mode`: 즉각적인 응답이 필요한 간단한 작업에 권장.
    *   `Planning Mode`: 복잡한 로직 설계 시 사용(토큰 소모량 높음).

### 3️⃣ AI 협업 워크플로우 (과업 지시 3단계)
1.  **목표 설정 (Goal Setting):** 만들고자 하는 서비스나 기능의 상위 목표 정의.
2.  **작업 지시 (Instruction):** 환경, 범위, 제한 사항 등을 포함한 구체적인 프롬프트 작성.
3.  **검토 및 피드백 (Review):** AI의 결과물을 검증하고 지속적으로 개선 요청.

### 4️⃣ 데이터 직군 트렌드 및 역할 변화
*   **변화하는 경계:** AI 기술 장벽 하락으로 인해 `Analytics Engineer`와 같이 여러 직군의 역량을 아우르는 하이브리드 역할 부상.
*   **직군별 역할 비유 (건축):**
    *   **DE (Data Engineer):** 파이프라인 및 인프라 구축 (토목 및 뼈대 공사).
    *   **DA (Data Analyst):** 데이터 기반 인사이트 도출 및 시각화 (내부 인테리어).
    *   **DS (Data Scientist):** 머신러닝 모델링 및 예측 (특수 설계).
*   **핵심 역량:** 도메인 지식 이상으로 데이터에서 이상치를 발견하고 **"왜?"**라는 질문을 던져 비즈니스 임팩트를 만들어내는 **이슈 제기(Issue Raising)** 능력이 최우선.
